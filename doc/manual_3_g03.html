Various options and parameters of the ANT.G code are user-controlled through the job_name.ant input file. This file must be in the same directory as that of the GAUSSIAN03 input file (job_name.com). If there is no parameter file the program will run with the default values, which are given here. The same holds true for an empty file or a file only containing comments and/or blank lines. Remember that job_name; is specified in the title line of the job_name.com or job_name.gjf file, it does not take it from the command line!!.

Syntax rules:
  1. The order of statements is arbitrary.

  2. Comment lines start with the symbol "!".

  3. Empty lines are ignored.

  4. General syntax for real and integer variable assignment:
    KEYWORD = (value)
    where KEYWORD is always in uppercase. All allowed keywords are listed below along with the default value assigned to them.

  5. Logical variable syntax:
    KEYWORD
    If the keyword is specified, the variable is given the value .TRUE. The default value is .FALSE. Thus the logical variables act as switches turning on a specific feature. For example, the keyword MULLIKEN dumps the Mulliken population analysis into a file.
Calculation parameters:
  1. ALPHA (real)
    Mixing parameter for density matrix (typically 0.01 < ALPHA < 0.1).
    Default: ALPHA = 2.0D-2

  3. NPULAY (integer)
  4. DIIS convergence parameter (typically 1 < NPULAY > 10). Every NPULAY normal convergence steps a Pulay extrapolation kicks in.
  5. Default: NPULAY = 4
  1. PACC (real)
  2. Numerical integration maximum accuracy for density matrix.
  3. (It starts at larger values in the early cycles, reaching the selected accuracy towards the end)
  4. Default: PACC = 1.0D-7

  6. CHARGEACC (real)
    Total charge maximum accuracy.
    (It starts at larger values in the early cycles, reaching the selected accuracy towards the end)
    Default: CHARGEACC = 1.0D-5

  7. FERMIACC (real)
    Fermi level maximum accuracy (in eV).
    (It starts at larger values in the early cycles, reaching the selected accuracy towards the end)
    Default: FERMIACC = 1.0D-5

  9. FULLACC (logical)
  10. Turns on full accuracy from the first cycle.

  11. SELFACC (real)
    Self-energy accuracy (Dyson selfconsistency).
    Default: SELFACC = 1.0D-6

  12. NIP (integer) (real)
    Maximum number of points for integration on the real axis.
    It follows the rule: n=(NIP+1)/2 ; NIP=3 (n=2); NIP=7 (n=4) ; NIP=15 (n=8) ...
    Default: NIP = 127

  13. FMIXING (logical)
  14. Switch on Fock matrix mixing (instead of density matrix) for damping.

  15. DMIMAG (logical)
  16. Computes the density matrix by integration along imaginary axis instead of along a complex contour (default).
SMALL, SMALLD (real)
Parameters used for the identification of the crystallographic structure of the electrodes and for determining the Bethe lattice directions. The smaller, the more stringent on the atomic structure.  Increase their values if your electrodes are slightly disordered.
Default: SMALL = 0.1
Default: SMALLD = 0.1 (Angstrom)
  1. ETA (real)
    Small imaginary part of the Green's function. When both electrodes are set to GHOST this number might need to be no so small.
  2. Default: ETA = 1.0D-10

  3. SL (real)
    Convergence criterion for when to switch the electrodes self-energy on. These are connected when the standard GAUSSIAN03/09 SCF convergence for the density matrix (RMSDP) reaches RMSDP < SL, where the overall convergence criterion is set in the GAUSSIAN03/09 input file with the SCF option "scf(conver=n)".
    Remark: If SL = 0, the leads are only connected after the GAUSSIAN03/09 SCF calculation has converged (cycle=1000). This is only recommended for quick and dirty calculations.
    Default: SL = 1.0D-2

  4. FERMISTART (real)
    Starting value for Fermi level search (in eV). When reinitializing a calculation from a P.job_name.dat file it is overruled by the existing value in P.job_name;.dat.
    Default: FERMISTART = -5.0D0

  5. FIXFERMI (logical)
  6. The SCF procedure is carried out fixing the Fermi energy all along (gran canonical esemble). The value is fixed by FERMISTART. This is a fast option, but the charge fluctuates and the convergence is not easily achieved.
TYPE1 and TYPE2 (string)
Type of model for the semi-infinite electrodes.
When set to BETHE, the effective selfenergy is connected to the electrode.
When set to GHOST, nothing is connected. When both electrodes are set to GHOST the calculation is similar to the one performed by the original GAUSSIAN03/09 code on the input  cluster, but with all the convergence advantages of ANT.G.
When set to 1DLEAD, a 1D selfenergy is build out of the selected atoms (see NEMBED keyword). (You must be familiar with the requirements to build 1D self-energies in 1D systems.)
Default: TYPE1 = BETHE and TYPE2 = BETHE
  1. BLPAR1 and BLPAR2 (string)
    Parameters sets of the Bethe lattice for the first and second electrode, respectively (see available options in files BLxxx.dat).
    Default: BLPAR1 = Papacon and BLPAR2 = Papacon

  2. OVERLAP (real)
    Factor by which the overlap parameters of the Bethe lattice (when specified in the parameters set) are multiplied (0.0 =< OVERLAP =< 1.0).  When set to a negative value, an automatic procedure to include the overlap (even if not specified in the parameters set) is used. This is the best option although it slows down the calculation with the PGI compiler. Set it to 0.0 for faster calculations and compare. This parameter may need to be smaller than 1.0 to avoid unphysical density of states for the Bethe lattice when the overlap parameters are > 0.2.
    Default: OVERLAP = 0.0

  3. GLUE (real)
  4. Factor by which to multiply the selfenergy. Useful to detach in a controled way the system from the Bethe lattice.
  5. Default: GLUE = 1.0

  7. HTRANSM (logical)
    Compute Transmision matrix in hermitian form T = \Gamma_L^{1/2} G^\dagger \Gamma_R G \Gamma_L^{1/2}. The default is the easier non-hermitian form \Gamma_L G^\dagger \Gamma_R G which is considerably faster for large clusters. The diagonalization of both forms yield the same eigenvalues.
    Default: .false. (transmission matrix in non-hermitian form).

    8.
  8. NEMBED1 and NEMBED2 (integer)
    Number of atoms to be connected to a Bethe lattice branch. These atoms are not taken into account on computing the charge of the system when OVERLAP > 0.
    3D Default: NEMBED1 = All atoms in the outermost plane of first electrode and NEMBED2 = All atoms in the outermost plane of second electrode (this is typically a good choice). If you do not like this default, make sure that the atoms in the input file are ordered in a way that suits your needs.
    2D Default:  NEMBED1 = All atoms in the electrode 1 and NEMBED2 = All atoms in electrode 2. Since you do probably not want this default, you should specify these. Make sure that the atoms in the input file are ordered in a way that suits your needs.
    When 1DLEAD is selected as type of electrode (see TYPE keyword above), these numbers determine the atoms of the cell used to build the self-energy. Here the choice is the logical one if you are familiar with 1D systems. Make sure that the atoms in the input file are ordered in a way that suits your needs.
NATOMEL1 and NATOMEL2 (integer)
Number of atoms in first and second electrode. Only useful when the system is composed of a single element or one wants to select especific regions for analysis.
Default: NATOMEL1 = Number of atoms in first metallic electrode and
NATOMEL2 = Number of atoms in second metallic electrode.

Reinitialization options:
  1. When the file P.job_name.dat is present, the calculation is initialized from the density matrix stored in P.job_name.dat. It is advisable to include the keyword nodamp in the SCF options of the GAUSSIAN03 input line "scf=(nodamp)" and make sure that the standard orientation of the system is the same as the one of the density matrix stored in P.job_name.dat. You also need to add "nosymm" to the input file job_name.com as "scf(nodamp,nosymm)" if you want to start with an educated guess for magnetic systems with an unusual spin texture or apply a finite bias voltage. The "nosymm" keyword placed alone on the SCF especifications line (not in the scf brackets) is useful to prevent GAUSSIAN03/09 from rotating the system and is usually necessary for optimizations since GAUSSIAN03/09 likes to change the standard orientation from step to step.

  3. PFIX (logical)
  4. When present, part of a previously computed density matrix can be used.
  5. Syntax:
  6. PFIX
  7. 'P.old_job_name.dat'
  8. where 'P.old_job_name.dat' is the file containing the density matrix you want to use. This is useful, .e.g., to change the basis set on a restricted set of atoms while preserving the convergency already achieved in the rest of the system (see NFIX).
  9. Default: .false. (no supplementary matrix provided).

    11. NFIX (integer)
    Number of atoms NOT to be touched when using PFIX.
    Syntax:
    NFIX = N
    #atom1, #atom2, ...
    where the second line indicates the atoms to be preserved.
Bias and gate voltage related parameters:
  1. BIASVOLT (real)
    Bias voltage (in eV). It is recommended to perfom first a calculation at zero bias and then reinitialize from P.job_name.dat. Succesive calculations at different bias voltages can be perfomed starting from a previous calculation in the same way. You need to add "nosymm" to the SCF specifications line in  the GAUSSIAN03 input file as "scf(nosymm)". The potential drop profile can be finally read from the file V.job_name.dat where the atom on-site energies (averaged to all orbitals in the atom) are stored.
    Default: BIASVOLT = 0.0

  2. QEXCESS (real)
    The effect of a gate voltage can be simulated through the addition of positive charges in the GAUSSIAN03/09 input file. To compensate and keep overall charge neutrality, an equal number of electrons should be added. This can be acomplished with this keyword. (Of course one is free to violate charge neutrality if physically required.) It overrules the initial charge given to GAUSSIAN03/09 through its input file, but it is recommended these two quantities to be equal for a more efficient calculation.
    Default: QEXCESS = 0.0
Spin related parameters:
  1. SWOFFSPL (real)
    Convergence criterion for switching off spin locking in a spin unrestricted calculation (default). Thus in a spin-unrestricted calculation the total spin of the device is kept fixed to the value $(M-1)/2$ given by the multiplicity $M$ in the GAUSSIAN03/09 input file until a certain convergence is reached: RMSDP < SWOFFSPL. (See also NSPINLOCK below)
    Advice: SWOFFSPL < SL
    Default: SWOFFSPL = 1.0D-3

    2.
  2. NSPINLOCK (integer)
    Number of steps with spin locking on. It supercedes the SWOFFSPL criterion  when >= 0 (see keyword SWOFFSPL above)
    Default: NSPINLOCK = 0.

    3.
  3. UD, DU, DD (logical)
    Magnetization direction of the Bethe lattice electrodes when these are magnetic. U stands for  "up" and and  D for "down". Useful after editing the density matrix to create an educated guess for a new calculation.
    Default: .false. (both electrodes "up").

    4.
  4. MRSTART (integer)
    Useful to create an educated guess for a new calculation with an unusual spin texture starting from a precomputed density matrix. In particular, it reverses the magnetization direction for all atoms with numbers >= MRSTART. You need to add "nosymm" to the SCF specifications line in  the GAUSSIAN03/09 input file as "scf(nosymm)".
    Default: MRSTART = 0 (No effect).

    5.
  5. SPINEDIT (logical)
    Useful to create an educated guess for a new calculation with an unusual spin texture starting from a precomputed density matrix. In particular, it manipulates the magnetization direction (spin) of individual atoms. You need to add "nosymm" to the SCF specifications line in  the GAUSSIAN03/09 input file as "scf(nosymm)".
    Syntax:
    SPINEDIT
    #natoms
    #atom1, #atomspin1
    #atom2, #atomspin2
    ...
    where #natoms is the number of atoms where the spin is altered. The list below #natoms gives the atom numbers and the atom spins for the atoms to be altered where the atom spin can either be maintained (1), reversed (-1), or erased (0).
    Default: .false. (off)

  6. SOC (logical)
    Activates atomic spin-orbit coupling. When true, the transmission and DOS are computed adding the spin-orbit coupling to the solution converged in the absence of spin-orbit coupling. As expected, in magnetic systems the spatial orientation determines the result. To be used in conjunction with global or per-atom SOC keywords: SOC_VAL_*, SOC_FAC_*, SOCVALATOM, and SOCFACATOM.
    Default: .false. (off)

  7. RCUT (real)
    Global cutoff (in Bohr) of the modified Yukawa screening potential. A value of zero means no screening (Z/r) and a large value means full screening (1/r). A reasonable value for this parameter is around two atomic radius.
    Default: RCUT = 2.0 bohr

  8. SOC_FAC_P (real)
    Global atomic SOC multiplicative factor for p (L=1) shells. The radial SOC matrix elements for all atoms with p shells in their basis sets are multiplied by this parameter. Useful for single-element systems where valence shell basis sets lack adequate nodal structure in their radial parts. The default value is 1.0, but higher values can be needed. These can be found by fitting to the actual band structure of the bulk system. A value of exactly 1.0 should be OK for basis sets of high enough quality. These factors are used in combination with a global value of RCUT. For further details, please see S. Pakdel, M. Pourfath, and J. J. Palacios, Beilstein J. Nanotechnol. 9, 1015Â–1023 (2018).
    Default: SOC_FAC_P = 1.0

  9. SOC_FAC_D (real)
    Global atomic SOC multiplicative factor for d (L=2) shells. The radial SOC matrix elements of all atoms with d shells in their basis sets are multiplied by this parameter. See keyword SOCFAC_P for more details.
    Default: SOC_FAC_D = 1.0

  10. SOC_FAC_F (real)
    Global atomic SOC multiplicative factor for f (L=3) shells.. The radial SOC matrix elements of all atoms with f shells in their basis sets are multiplied by this parameter. See keyword SOCFAC_P for more details.
    Default: SOC_FAC_F = 1.0

  11. SOCFACATOM (logical)
    Per-atom, per angular momentum shell (L= 1, 2, or 3), atomic SOC multiplicative factors and per-atom cutoff radius of the modified Yukawa nuclear screening potential. The radial SOC matrix elements of the listed atoms are multiplied by the hereby given SOC_FAC_* and RCUT parameters. See keywords SOC_FAC_* and RCUT for further details.
    Syntax:
    SOCFACATOM
    #natoms
    #atom1, #atomsocfac_p1, #atomsocfac_d1, #atomsocfac_f1, #atomrcut1
    #atom2, #atomsocfac_p2, #atomsocfac_d2, #atomsocfac_f2, #atomrcut2
    ...
    where #natoms is the number of atoms where the p, d and f shell SOC_FAC_* values and cut-off radius are altered. The list below #natoms gives the atom numbers and a set of four user-specified atom SOC_FAC values (Default parameter values: 1.0 each) and the per-atom cutoff of the modified Yukawa nuclear screening potential (Default parameter value: 2 Bohr).
    Default: .false. (off)

  12. SOC_VAL_P (real)
    Global atomic SOC value for p (L=1) shells. When different from zero, the angular SOC matrix elements of all atoms with P shells in their basis sets are set to this value. Useful for single-element systems in which the empirical values are available. Works best for GTO or CGTO minimal basis sets with only a single L=1 shell. It superseeds SOC_FAC_P when finite.
    Default: SOC_VAL_P = 0.0 eV

  13. SOC_VAL_D (real)
    Same as SOC_VAL_P, but for d (L=2) shells.
    Default: SOC_VAL_D = 0.0 eV

  14. SOC_VAL_F (real)
    Same as SOC_VAL_P, but for f (L=3) shells.
    Default: SOC_VAL_F = 0.0 eV

  15. SOCVALATOM (logical)
    Per-atom atomic SOC values for L=1, 2, and 3 shells. Useful for multi-element systems where the empirical values of the SOC values are available. Works best for GTO or CGTO basis sets with single p and/or d shells and/or f shells..
    Syntax:
    SOCVALATOM
    #natoms
    #atom1, #socvalatom_p1, #socvalatom_d1, #socvalatom_f1
    #atom2, #socvalatom_p2 , #socvalatom_d2 , #socvalatom_f2
    ...
    where #natoms is the number of atoms where the empirical p, d and f shell SOC values are set. The list below #natoms gives the atom numbers and a set of three user-specified atom SOC values (Default parameter values: 0.0 eV each). It superseeds SOC_FAC_* and SOCFACATOM when finite.
    Default: .false. (off)

  16. ROT (logical)
    Activates atomic spin rotations. When true, the transmission and DOS are computed after rotating the spins of the spin-unrestricted converged solution [see C. R. Phys. 17, 406-429 (2016)]. As expected, in magnetic systems the spatial orientation determines the result. To be used in conjunction with global or per-atom ROT options. See keywords THETA, PHI, and SPINROTATOM.

  17. THETA (real)
    Global polar angle (in degrees) of the spin rotation operation applied uniformly to all atomic spins in the device. Valid input angle runs from -180.0 to 180.0 degrees.
    Default: THETA = 0.0 degrees

  18. PHI (real)
    Global azimuthal angle (in degrees) of the spin rotation operation applied uniformly to all atomic spins in the device. Valid input angle runs from -360.0 to 360.0 degrees.
    Default: PHI = 0.0 degrees

  19. SPINROTATOM (logical)
    Per-atom atomic spin rotation angles. Useful in systems where only a subset of atoms are magnetic and a select number of atomic spins are to be rotated.
    Syntax:
    SPINROTATOM
    #natoms
    #atom1, #atomtheta1, #atomphi1
    #atom2, #atomtheta2, #atomphi2
    ...
    where #natoms is the number of atoms where the atomic spins are rotated. The list below #natoms gives the atom numbers and a set of two user-specified atom theta and phi angles. Valid input atomtheta angles run from -180.0 to 180.0 degrees and -360.0 to 360.0 degrees for the atomphi angles. (Default parameter values: 0.0 degrees each).
    Default: .false. (off)

Output parameters
  1. ESTEP (real)
    Energy step (in eV) for printing out transmission and DOS.
    Default: ESTEP = 1.0D-2

    2.
  2. EW1 and EW2 (real)
    Define an energy window for printing out transmission and DOS.
    Default: EW1 = -3.0D0 and EW2 = 3.0D0.

    3. DOSENERGY     (real)
    Energy at which to evaluate the DOS projected on atoms when keyword MULLIKEN on.
    Default: 0.0 (Fermi Energy).

  3. LDOS_BEG and LDOS_END (integer)
    Atoms numbers onto which the local density of states is evaluated.
    Default: LDOS_BEG = 1 and LDOS_END = 0 (LDOS evaluation off).

    4.
  4. POL (logical)
    Evaluation of the polarization. It gets printed along the transmission in the corresponding T.jobname.dat file
    Default: .false.

  5. NCHANNELS (integer)
    Number of eigenchannles to print out in T.job_name.dat. If NCHANNELS > 0 then the tranmission matrix (calculated in the non-hermitian or hermitian form, see keyword HTRANSM) is diagonalized and the NCHANNELS largest eigenvalues (i.e. the eigenchannel transmissions) are printed.
    Default: NCHANNELS = 0 (eigenchannel calculation off).

    6.
  6. LEADDOS (logical)
    Prints out the DOS of the Bethe lattice. Output files: Lead1DOS.dat, Lead2DOS.dat
    Default: .false. (off).

    7.
  7. RTM_BEG and RTM_END (integer)
    Perform eigenchannel analysis with reduced transmission matrix [see PRB 73, 075429 (2006)] on atoms from atom RTM_BEG to atom RTM_END Default: RTM_BEG = 1 and RTM_END = 0 (eigenchannel analysis off).

  8. MULLIKEN (logical)
    Dumps the Mulliken population analysis in the file Q.job_name.dat. The DOS projected on atoms at the Fermi energy (or at the energy selected by DOSENERGY) is also dumped when LDOS_BEG < LDOS_END.
    Default: .false. (not print).
Options for electronic correlations
    MOLBLOCKS
    This keyword specifies a block of atomic orbitals (typically defining a molecule) for further treatment beyond DFT to open a charge gap.
    Syntax:
    <num_corrblocks>
    <ao_beg> <ao_end>
    where <num_corrblocks> is the number of correlated blocks within the device region. For each correlated block a line "<ao_beg> <ao_end> " has to be specified where <ao_beg> is the number of the first atomic orbital and <ao_end> is the last atomic orbital of the correlated block.

  1. UPLUS (real)
    Charging energy of the correlated block (molecule) defined by MOLBLOCKS (same for all blocks at this point).
    Default: UPLUS = 0.0

    2. CORRBLOCKS
    This keyword specifies a block of atomic orbitals (usually a d- or f-shell of an atom) for further treatment beyond DFT taking into account local electronic correlations.
    Syntax:
    <num_corrblocks>
    <ao_beg> <ao_end> <U> <J>
    ...
    where <num_corrblocks> is the number of correlated blocks within the device region. For each correlated block a line "<ao_beg> <ao_end> <U> <J>" has to be specified where <ao_beg> ist the number of the first atomic orbital and <ao_end> is the last atomic orbital of the correlated block. <U> is the direct Coulomb repulsion and <J> Hund's rule coupling for the correlated block.

    DFT+U (logical)
    Requests a DFT+U calculation where a Hubbard interaction term (U and J) is put onto each correlated block which is treated within Hartree-Fock.

    HYBFUNC (logical)
    Requests the calculation of the hybridization function for each correlated block which serves as an input for an impurity solver calculation. A mesh file "mesh.dat" needs to be present specifiying the energy points for which to calculate the hybridization function.

    PORTHO (logical)
    Requests to perform a "projective orthogonalization" prior to calculation of the hybridization function. Thereby the device region is orthogonalized in a Gram-Schmidt like manner that preserves the original correlated subspace composed of all correlated blocks.

    DIAGCORRBL (logical)
    Prior to calculation of the hybridization function the Hamiltonian of each correlated subspace is diagonalized. This is useful in case of strong crystal-field splitting leading to large off-diagonal elements in the hybridization function. Using this option the crystal-field basis is found, and hence off-diagonal elements of the hybridization function are minimized. This option is also useful in case that the correlated subspace is described by more than one set of orbitals (e.g. double-zeta basis). In this case, the diagonalization yields the low-energy subspace of the correlated shell.
